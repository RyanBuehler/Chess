# v3-zenith-w8 (Chained Means-End Goals) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a fully-segregated `emergent_chained` self-play mode (v3-zenith-w8) where each side chains discovered cluster sub-goals — greedy `curriculum_weight(g)·v_goal(s,g)` selection, achieve-or-expire reassignment, smooth α-schedule win-apex — leaving v2-halcyon (`emergent`) byte-reproducible.

**Architecture:** v3 chaining LOGIC lives in dedicated new modules (`chessrl/selfplay/meansend_chained.py`); v2's `concurrent.py` means-end functions are untouched. Deep shared infra (the means-end MCTS leaf-blend, cluster HER, the replay buffer) gains only **additive, v2-inert hooks** (a per-tree α override; a HER look-ahead cap that defaults to v2 behavior), each guarded by a regression test proving v2's path is unchanged. Dispatch is by `goal_mode == "emergent_chained"` in the worker, training loop, and eval factory. The net, goalspace, vector buffer, dual-head trainer, and the α=0 eval player are reused unchanged.

**Tech Stack:** Python, numpy, python-chess, pytest. CPU/CUDA torch via the existing `VectorGoalNetEvaluator`.

## Global Constraints

- **SEGREGATION (hard):** v2-halcyon (`goal_mode: emergent`) must remain independently runnable and **byte-reproducible**. Every shared-file change is additive and defaults to exactly the v2 behavior; v2 code never sets the new hooks. Task 9 is the regression gate proving this.
- Goal scale fixed: `goal_window = 8` (isolates goal-scale vs a future `v3-zenith-w16`).
- v3 selection score: `score(g) = curriculum_weight(g) · v_goal(s, g)`, `curriculum_weight(g) = novelty_beta·novelty(g) + gamma_winvalue·max(0, win_value(g))` (the existing `ClusterCurriculum` weight; never zero — novelty floor).
- α-schedule: `α(s) = meansend_alpha · (1 − clip(|v_win(s)| / win_ramp, 0, 1))`, and `α = 0` when `ply ≥ ply_cap − endgame_margin`. No discrete win-switch.
- Reassign on **achieved (live)** OR **expired (`goal_window` plies)**.
- Defaults (new `GoalConfig` fields): `goal_select_temp = 0.5`, `win_ramp = 0.6`, `alpha_resign_gate = 0.05`, `endgame_margin = 20`. `meansend_alpha` (0.25) is the α_max.
- Windows venv: run tests with `.venv\Scripts\python.exe -m pytest`. Stage only named files; never `git add -A`. Push after each task.
- Eval is identical to v2 (α=0 terminal pursuit on `win_vector`); `emergent_chained` routes to the existing `VectorGoalMCTSPlayer`.

```python
"""v3-zenith: chained means-end self-play. ENTIRELY SEGREGATED from v2's
selfplay/concurrent.py — v2 (goal_mode 'emergent') never imports this module.

Each side always holds an active cluster sub-goal, re-selected on achieve-or-expire
via a greedy curriculum_weight(g)*v_goal(s,g) score; goal-influence (the means-end
leaf alpha) fades to 0 as the position becomes decisive (alpha_schedule). 'Win as
apex' emerges from alpha->0; there is no discrete terminal goal during play."""
from __future__ import annotations

import numpy as np
import chess

from chessrl.chess_env.encoding import encode_board, to_model_input


def alpha_schedule(v_win: float, alpha_max: float, win_ramp: float,
                   ply: int, ply_cap: int, endgame_margin: int) -> float:
    """Goal-influence weight for the means-end leaf. Full (alpha_max) in unclear
    positions, -> 0 as |v_win| -> win_ramp (decisive either way) or near ply_cap."""
    if ply >= ply_cap - endgame_margin:
        return 0.0
    decisiveness = min(1.0, abs(float(v_win)) / max(win_ramp, 1e-6))
    return float(alpha_max) * (1.0 - decisiveness)


def select_next_goal(board: chess.Board, goalspace, curriculum, evaluator,
                     goal_cfg, rng) -> tuple[int, np.ndarray]:
    """Greedy state-dependent next sub-goal: argmax/softmax over
    curriculum_weight(g) * v_goal(board, centroid_g). Epsilon-explore injects a
    uniform-random cluster. Requires goalspace.ready (caller guards)."""
    K = goalspace.n_clusters
    if rng.random() < getattr(goal_cfg, "epsilon", 0.0):
        c = int(rng.integers(K))
        return c, np.asarray(goalspace.centroids[c], np.float32)
    cents = np.asarray(goalspace.centroids, np.float32)            # (K, d)
    planes = to_model_input(encode_board(board))
    planes_batch = np.repeat(planes[None, ...], K, axis=0)
    deadlines = np.full(K, goal_cfg.goal_window, np.float32)
    _, _v_win, v_goal = evaluator.evaluate_planes(planes_batch, cents, deadlines)
    w = np.array([curriculum.weight(c) for c in range(K)], np.float64)
    score = w * np.asarray(v_goal, np.float64)
    temp = max(float(goal_cfg.goal_select_temp), 1e-6)
    logits = score / temp
    logits -= logits.max()
    p = np.exp(logits); p /= p.sum()
    c = int(rng.choice(K, p=p))
    return c, cents[c].astype(np.float32)
```

```python
from dataclasses import dataclass

@dataclass
class _ChainSideGoal:
    active_cluster: int
    active_vec: np.ndarray
    start_ply: int
    start_emb: np.ndarray      # frozen-encoder embedding of the state where this goal began
    explore: bool = False

def assign_chain_goal(board, ply, goalspace, curriculum, evaluator, goal_cfg, rng) -> _ChainSideGoal:
    eps = getattr(goal_cfg, "epsilon", 0.0)
    explore = rng.random() < eps
    c, vec = select_next_goal(board, goalspace, curriculum, evaluator, goal_cfg, rng)
    start_emb = np.asarray(evaluator.embed_boards([board])[0], np.float32)
    return _ChainSideGoal(c, vec, ply, start_emb, explore)

def goal_achieved_live(side, board, goalspace, evaluator) -> bool:
    emb = np.asarray(evaluator.embed_boards([board])[0], np.float32)
    return bool(goalspace.achieved((emb - side.start_emb).astype(np.float32), side.active_cluster))

def maybe_reassign_goal(side, board, ply, goalspace, curriculum, evaluator, goal_cfg, rng):
    expired = (ply - side.start_ply) >= goal_cfg.goal_window
    achieved = goal_achieved_live(side, board, goalspace, evaluator)
    if expired or achieved:
        return assign_chain_goal(board, ply, goalspace, curriculum, evaluator, goal_cfg, rng)
    return side
```

(`select_next_goal`'s epsilon branch and `assign_chain_goal`'s `explore` flag both consult `epsilon`; keep `assign_chain_goal` as the single entry that stamps `explore` and `start_emb`.)
